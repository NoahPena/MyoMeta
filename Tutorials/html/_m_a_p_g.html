Meta App Programming Guide

Meta App Programming Guide

This guide introduces how to build apps for Meta.

This is a guide to making Meta applications from a coding perspective. To learn about designing Meta user interfaces, see the Meta Human Interface Guidelines.

The building blocks of Meta SDK Guide app

Coding Convention

Meta SDK follows coding conventions based on Unity and C# norms.

  • Unity Gems has numerous explanations for mastering the nuances of Unity programming.

The Unity GameObject hierarchy

  • A Unity scene is composed of GameObjects.
  • GameObjects have a name that shows up in the Hierarchy panel and at the top of the inspector when they are selected.
  • GameObjects have a Transform component with their position, rotation and scale properties.
  • Transform also determines if a GameObject has a parent GameObject above it in the hierarchy.
  • GameObjects at the top of the hierarchy have no transform parent.

To learn more about Unity, see Learning Unity Guide.

MetaBody Component Guide

The MetaBody Inspector

To make a GameObject responsive to Meta interactions:

  1. In Hierarchy, select the GameObject.
  2. In Inspector, click Add Component.
  3. Click MetaBody.

A MetaBody component is added to the GameObject and it shows up in the inspector list. GameObjects with many components may require you to scroll down the Inspector panel to see them all.

You can collapse some components that you don't edit much to keep MetaBody and other components you need to edit visible.

To learn more about what MetaBody can do, see the MetaBody API reference.

When you enable checkboxes on MetaBody for things like MetaBody.grabbable and MetaBody.moveable, Meta Gesture Manager automatically works with the objects and allows you to do those things without manually adding any additional components.

Enabling or disabling some MetaBody properties may add or remove components at run-time.

Property Component Added
MetaBody.Jaysify VectorMeshObject
MetaBody.Feedback HighlightableObject
MetaBody.CanvasTarget Meta.CanvasTarget
MetaBody.MarkerTarget Meta.MarkerTarget

Visuals lets you customize the appearance of your object.

Jaysify automatically applies a holographic material with a procedurally animated vector outline.

Feedback provides visual and audio feedback cues when your hands near gesturally interactive bodies.

MetaBehaviour scripts handle events generated by MetaBody interactions.

MetaBehaviour Scripting Guide

MetaBehaviour inherits from MonoBehaviour and adds augmented reality behaviour for use with Meta.

To make a script work with Meta, change the : MonoBehaviour inheritance of the class to : MetaBehaviour instead. MetaBehaviours add themselves to MetaManager.MetaObjects list.

MetaBehaviours require a MetaBody to initiate their events and when you add a MetaBehaviour to a game object, a MetaBody will be automatically added if none is present.

MetaBehaviour's handler methods can be implemented by your scripts.

Event Method Purpose
MetaBehaviour.OnGrab() Handles events when user closes hand on a MetaBody.grabbable game object.
MetaBehaviour.OnHold() Handles when a MetaBody.grabbable game object has been/is held.
MetaBehaviour.OnRelease() Handles when a user opens their hand after grabbing MetaBody.grabbable.
MetaBehaviour.OnClick() Unity UI message method for mouse clicks on the object, in Meta it also handles a clicking fingertip touch.
MetaBehaviour.OnHover() Handles a fingertip pointing at the object.
MetaBehaviour.OnTouchEnter() Handles when a finger touches the MetaBody.touchable object.
MetaBehaviour.OnTouchHold() Handles when a finger stays inside the MetaBody.touchable object.
MetaBehaviour.OnTouchDwell() Handles when a finger stays inside the MetaBody.touchable object for more than the 2 second dwell threshhold.
MetaBehaviour.OnTouchExit() Handles when a finger ceases being inside the previously touched MetaBody.touchable object.
MetaBehaviour.OnPinch() Handles events when user pinches on a MetaBody.pinchable game object.
MetaBehaviour.OnPinchHold() Handles when a MetaBody.pinchable game object has been/ is pinched.
MetaBehaviour.OnPinchRelease()Handles when a MetaBody.pinchable game object has been released.
MetaBehaviour.OnGazeDwellExit() Handles events when a user's gaze has dwelled and then leaves the target's collider.

For more details, see the MetaBehaviour class scripting reference.

Canvas Guide

We use our own unique surface tracking algorithm to recognize surfaces and attach digital content onto them.

Surfaces

A surface is defined as an object that can be augmented using the Meta SDK. The surfaces fully supported by the current version are the tops or sides of small boxes, thick books, etc, sitting on a table or in any stationary setting. They will have to be part of the environment and not move freely.

Materials of the surface will impact performance. Limitations are dark and/or shiny surfaces will not perform as well as lighter and matte ones.

Canvas

A canvas is defined as a digital placeholder that can hold augmented information. Digital canvases are attached to real-world surfaces. Any MetaBody can be automatically

How To

CanvasTarget

The current version supports attaching canvases to one surface at a time. The way to do it is the following:

  1. The developer indicates which MetaBodys will be latched to a canvas by checking the “Canvas Target” checkbox at the inspector for each MetaBody (alternatively you can enable MetaBody.CanvasTarget in code).
  2. The user wearing Meta 1 looks at the desired surface.
    Desired surface
  3. The MetaBody temporarily disappears and a cross appears indicating that this surface is being analyzed (pre-lock mode).
    Tracking surface
  4. If the system is confident that this is a suitable surface to track, the cross will disappear and the MetaBody will appear on the selected surface (post-lock mode). Otherwise, the cross will persist (see do’s and don’t’s about the best surfaces to track).
    Successful Surface Tracked
  5. Until the surface goes out of the field of view of the user, the MetaBody will remain locked to that surface. Otherwise, it will automatically get detached and the cross will re-appear.
  6. In order to unlock the object from the canvas, you can either swipe with your hand on the air, or look away from the surface.

Do's

  • Place surfaces on tables or stationary locations.
  • Prefer boxes with dimensions between 0.5-1 feet. (15-30cm), or thick books >1 inch of thickness (2.54cm).
  • Maintain a distance of <80cm from your surfaces.
  • Make sure the surfaces you desire to track are parallel or perpendicular to gravity.

Don't's

  • Do not hold and/or move surfaces.
  • Do not use very small surfaces or very large surfaces
  • Do not exceed the distance 80cm from the surfaces.

Markers Guide

April Tag Scientific White Paper

Meta supports April Tag fiducial markers with unique ID numbers. You can print the markers out by finding the ‘AprilTagMarkers0-11.pdf’ file located inside you Meta project in the Meta folder

MetaBody.MarkerTarget

Enabling a game object's MetaBody.MarkerTarget will add a MarkerTarget component to it assigned to a specified target marker ID. The current version of Meta SDK limits it to detecting 10 markers simultaneously.

MetaBody.markerTargetPersistent

Enabling a MetaBody.markerTargetPersistent will ensure the objects placed onto the specific marker ID will have a persistent placement on the marker, and will remember the marker it was placed on when the scene is loaded again.

MetaBody.markerTargetPlaceable/ MetaBody.markerTargetPlaceableHighlight

Enabling a MetaBody.markerPlaceable will allow you to pick MetaBody game objects and drop them onto a marker without signing a preset marker ID attached to the object. MetaBody.markerTargetPlaceableHighlight will make the MetaBody game object highlight when placed on a marker.

How To

MetaBody.MarkerTarget

  1. The developer indicates which MetaBodys will be a Marker Target object in the inspector window by checking off MarkerTarget (alternatively you can enable MetaBody.MarkerTarget in code).
  2. The user wearing Meta 1 looks at the desired marker ID
  3. MarkerTarget game object will latch onto the preset marker with the correct ID.

Limitations

The current Marker Target has a list of known limitations that will affect performance,

  • Use of 10 Markers at a time
  • Marker has to be fully facing the camera, or the camera will lose track (No folds or extreme angles)
  • The Field of View for Marker Target objects differ from the full Field of View for depth camera in MetaWorld he camera around the marker. It enables translation around a designated localized marker.

Common Functionality Patterns

There are some recurring patterns of functions developers want to implement in AR Apps.

Switching Scenes

When you build a Meta app with several scenes in it, you can switch scenes gracefully by calling MetaSpace.LoadScene or MetaSpace.LoadSceneAdditive instead of directly calling Application.LoadLevel. The MetaSpace version will display a LOADING indicator to reassure users to be patient during transitions between scenes instead of quitting what they think is a frozen app.

Put some objects on a table

  • Place a MetaBody.CanvasTarget or MetaBody.MarkerTarget on an object to attach it to a table.

Talking to Web APIs

Many Apps talk to Web Apps and need to be able to send and receive web requests.

  • Create a WWW request object.
  • Use StartCoroutine("RequestMethod"); to wait for an IEnumerator RequestMethod() with the reply (you can learn about Coroutines at Unity Gems).
  • If you need to craft a header, note that WWW.header is read-only and you need to pass the header data into the request separately.
  • MiniJSON and FullSerializer are included utilities for serializing and deserializing JSON data to and from web requests. MiniJSON serializes into a series of IList and IDictionary objects, whereas FullSerializer is able to serialize into a multitude of different formats.

Integrate a third-party vision library

To integrate a third-party vision library, there are two main options:

  • Integrate the library via compiling it as a C# or C++ plugin.
  • Implement the DepthSense SDK to access the RGB and depth camera feeds concurrently.

Namespaces

Meta classes are in the Meta namespace. Each application should have its own namespace.

Namespaces had some bugs in earlier versions of Unity that prevented some from using them, but they are now fully functional and required for applications that will interoperate in the MetaSpace in order to avoid causing conflicts with each other.

Programming Interactions

Distance Thresholds

When using distance threshholds to determine if a gesture is intended for a target object, 0.1m is the usual distance between gesture and target within which gestures can be accepted.

Magnetic Thresholds

A magnetic threshhold is one which has an attractive force against escaping it once it snaps on. This is based on the principle of hysteresis - this means that the state of something in the past changes its state in the future to resist change. This can be useful in a number of situations:

  • When grabbing an object from a place it is attached to.
  • When moving an object between two difference distance thresholds such as different spaces in a grid, the boundaries between them can result in jittering back and fourth between one side and the other of the boundary.

Time Threshholds

Time thresholds start a timer before something will happen. If dwell feedback is enabled, visual feedback of a circle drawing in lets the user know when to expect something to happen. Use Dwell by default, which has a 2 second threshhold, unless some other threshhold is needed.

Keyboard Shortcuts Guidelines

When designating keyboard shortcuts that do things in your applications, several considerations apply:

  1. Don't use the Function keys F1 through F8, which are reserved for SDK's Keyboard Shortcuts.
  2. Don't directly use Input.GetKey accessors without first checking with MGUI to see if a text input box is selected. If you get keypresses while a user is typing, typing into the text input will trigger your keyboard shortcuts.
  3. Check that MetaKeys.typing is false before responding to key input.
