Meta Version Control System Guide

Unity projects require special version control processes to handle not only source code but also many other kinds of assets.

There are a variety of strategies for doing version control with Unity:

  • Unity Asset Server: Fully integrated beginner-friendly user interface built into Unity editor (requires a Team License). No branching, but large binary art isn't a problem. Sometimes throws strange errors that require re-pulling a fresh copy of the project.
  • Git with Bitbucket: Powerful branching features with a learning curve for those new to Git. Requires .gitignore settings to tell Git to not version Library and Temp folders. Supports Jira for issue integration with your code commits/pushes.
  • Perforce: Integrated client built into Unity, but Perforce hosting services like Assembla choke on uploads of large binary file projects. Amazon S3 instances of Perforce are standard practice in the game industry if you have Perforce know-how to administer them.
