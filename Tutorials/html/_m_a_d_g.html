Meta Documentation Guide

    /// <summary>Summary comments are your friend.</summary>

This guide explains how Meta documentation was created in order to help you produce documentation for your own Meta App projects.

You can use a combination of Doxygen, C# XML comments and Markdown files to produce complete documentation for your Unity projects.

This is especially important if you produce a project that is an asset package for sharing with other Meta and Unity developers. Producing great documentation will help people use your package to produce great results.

What is Doxygen?

Doxygen is a tool for auto-generating a cohesive HTML documentation App like this one based on specially encoded comments included in your source code itself, so that documentation is maintained as the code is written rather than being a separate thing that quickly goes stale and de-synchronizes from the code that it is meant to describe.

XML Comments

The first step is to write triple slashes in the line above a class or member declaration. In Visual Studio it will auto-insert this:

    /// <summary>
    /// MemberName
    /// </summary>

Add these comments above each documented class and member in your source code itself. These comments get parsed by Doxygen and turned into searchable, indexed help pages.

How to include code blocks in XML comments

To include XML comments in your .cs code including example code showing how it works:

    /// <summary>
    /// The one sentence explanation of the member
    /// </summary>
    /// <remarks>
    /// The long-winded explanation of the member.
    /// </remarks>
    /// <example>
    /// Some text introducing the example code snippet.
    /// <code>
    /// ExampleMethodName("The example argument");
    /// </code>
    /// </example>
    public void ExampleMethodName {
        // actual implementation code
    }

Markdown Usage

Doxygen has Markdown support. For reference, see:

  • Doxygen Markdown Support Guide

Markdown editors help with editing. Some that are available are:

  • MarkdownPad 2: Free version available. Supports tabbed editing and live previews.
  • Markdown Mode: Visual Studio plugin that adds syntax highlighting for Markdown.

Grouping pages hierarchically

Markdown allows you to group related pages in a hierarchical manner. For instructions on how to do this, see:

  • Doxygen Grouping Guide
  • Stack Overflow: How to Use Markdown Pages and Groups

Documentation Writing Style Guide

  • Concision: Never use a paragraph where a sentence will do. Never use a sentence where a phrase will do. Never use a phrase where a word will do.

Documentation FAQ

What is the relationship between Doxygen codes in comments above each class and member in your .cs files and the main Doxygen help .cs and .md markdown files you choose to include in your Doxygen settings?

Each triple slash comment block you write above a class or member is used to generate the Classes index and pages that make up the core Doxygen documentation.

In addition to that, it is helpful to provide guide material explaining how to get started and supplementary tutorials and instructions.

To add such supplementary pages, you need to use Markdown files.

You should have a master ProjectNameHelp.cs file that includes a single block of comments telling Doxygen what to put on your main page and how to hierarchically place the pages and subpages.

The contents of the pages and subpages should be written in your MainPage.md file, broken up by:

@page PageName Page Title
